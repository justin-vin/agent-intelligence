Recipes

Packaged intelligence that compounds

A skill is a reusable unit of intelligence. It packages domain knowledge, decision logic, and workflow steps into something an agent can execute reliably. Skills are how intelligence becomes tangible — how the abstract craft of "make good decisions" turns into "handle this specific kind of work, well, every time."

The recipe model

The most natural path from problem to skill is the recipe model. You do something manually — process a batch of applicants, prep for a recurring meeting, triage an inbox. The agent watches. After seeing you do it two or three times, it recognizes the pattern.

"You've done this three times. The steps are: pull new applications, screen against these criteria, draft assessments, flag top candidates. Want me to handle this next time?"

One click. Now it's a recipe. The first time cost you 20 minutes. Every subsequent time costs you 10 seconds of review. That's the value proposition: intelligence that learns from your behavior and packages it for reuse.

Do something twice, teach it once, never do it again.

The skill lifecycle

1. Discover

Notice the pattern

Every skill starts as a repeated action. Someone does the same kind of work multiple times. The pattern might be obvious (weekly report generation) or subtle (the way you handle a specific type of customer email). Discovery is about recognizing that a human is doing something an agent could learn.

2. Package

Encode the intelligence

Turn the pattern into a skill: define the trigger, the inputs, the steps, the decision points, the output format. This is the craft — capturing not just what someone does but why they do it, what judgments they make, what trade-offs they consider.

3. Ship

Make it available

A skill that exists but nobody uses is wasted intelligence. Skills need to be discoverable — surfaced at the right moment, described clearly, easy to activate. The best skill is one the user doesn't even know exists until it appears exactly when they need it.

4. Iterate

Improve from usage

Every execution is feedback. Did the user accept the output or modify it? Where did they intervene? What edge cases appeared? Skills should get better over time — not just from explicit improvement but from implicit learning through use.

Design principles

  • Composable: Skills should combine. A "meeting prep" skill that uses the "email summary" skill and the "calendar context" skill is more powerful than three separate tools. Design for composition, not isolation.
  • Context-aware: A skill that ignores user preferences is just a template. Good skills adapt — to the user's style, the recipient, the time of day, the urgency. Context makes a generic skill feel personal.
  • Testable: Every skill should have clear inputs and expected outputs. If you can't test it, you can't improve it. If you can't improve it, it'll rot.
  • Transparent: Users should understand what a skill does before they run it, and see what it did after. Black-box automation erodes trust. Visible reasoning builds it.
  • Graceful: When a skill hits an edge case it can't handle, it should stop and ask — not forge ahead with bad output. The cost of asking is low. The cost of wrong output is high.

Skills as the first-run lever

The first time someone uses Sauna, they have no accumulated context. No preferences, no history, no recipes. The agent is smart but generic. This is the cold-start problem, and skills are the solution.

Out-of-the-box skills — meeting prep, email triage, research synthesis, morning briefing — give a new user immediate value before they've taught the system anything. They demonstrate what the platform can do. They create the "holy shit" moment that turns a trial into a conversion.

Every skill we ship is a bet: this is useful enough, for enough people, that it justifies its place in the default toolkit. The bar should be high. A few excellent skills beat a catalog of mediocre ones.

In practice

Meeting Prep

Know before you go

Sees a meeting on the calendar. Pulls attendee context from email history and CRM. Surfaces previous meeting notes. Flags open action items. Drafts an agenda if none exists. Delivered 15 minutes before the meeting starts.

Email Triage

Inbox zero, maintained

Processes new emails. Categorizes by urgency and type. Drafts responses for routine items. Flags anything that needs human judgment. Learns your response patterns — after 50 emails, the drafts start sounding like you wrote them.

Research Synthesis

From question to answer

Takes a research question. Gathers information from connected sources — documents, emails, web, databases. Synthesizes into a structured brief with sources cited. Distinguishes between facts and inferences. Flags confidence levels.

Batch Operations

Twenty tasks, one click

The superpower of skills: batch processing. Review 20 CVs against criteria. Send personalized follow-ups to 15 investors. Update 30 CRM records from meeting notes. What would take an afternoon takes minutes — with human review at the end.

Skills are where intelligence meets the user. All the reasoning, all the context, all the prompt engineering — it culminates in a skill that does something useful. Every skill we ship compounds the value of the platform. Every skill a user creates makes their agent more powerful. The goal isn't a smart agent. It's a skilled one.
